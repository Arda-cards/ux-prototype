# How to Create New Stories

This guide covers the conventions for creating Storybook stories in this project, including CSS theming, `App/Current` page stories, and component stories.

## Intent

This storybook needs to support 2 main goals:

1. Develoment of new Stories and components, using the existing design system in the application.
2. Evolution of the design system itself, including the evolution of components (through the [Canary Components Process](canary-components/how-to-extract-and-publish.mdx)) and the design tokens themselves.

The Current design system is imported from the [Arda frontend application](https://github.com/Arda-cards/arda-frontend-app), [Arda Application Preview](https://arda-mock-demo.vercel.app) in the [vendored](../../vendored/) directory. This is the one that needs to be used for any component or activity that is intended to be part of the final product.

Arda's Design System elements are defined in 3 versions:

| Story type                                                   | CSS to import                                          | Notes                              |
| ------------------------------------------------------------ | ------------------------------------------------------ | ---------------------------------- |
| Current-track component stories (`src/components/extras/`)   | None &#8212; uses the global default from `preview.ts` | Base theme is already loaded       |
| `App/Current` page stories (`src/dev-witness/`)              | `@/styles/extras/vendored-theme.css`                   | Must match the real app appearance |
| Migration-track component stories (`src/components/canary/`) | `@/styles/canary/globals.css` (when available)         | For migration-specific theming     |

### Example: `App/Current` Story with Theme Override

```tsx
// src/dev-witness/<section>/my-page.stories.tsx
import type { Meta, StoryObj } from '@storybook/react-vite';
import MyPage from '@frontend/app/my-page/page';

// Override base theme with vendored app styles
import '@/styles/vendored/globals.css';

const meta: Meta<typeof MyPage> = {
  title: 'App/Current/Section/My Page',
  component: MyPage,
  tags: ['app-route:/my-page'],
  parameters: {
    layout: 'fullscreen',
    appRoute: '/my-page',
    appComponent: 'app/my-page/page.tsx',
  },
  args: {
    pathname: '/my-page',
  },
};

export default meta;
type Story = StoryObj<typeof MyPage>;

export const Default: Story = {};
```

### Example: Current-Track Component Story (No Override Needed)

```tsx
// src/components/extras/atoms/my-atom/my-atom.stories.tsx
import type { Meta, StoryObj } from '@storybook/react-vite';
import { MyAtom } from './my-atom';

// No CSS import needed — uses global default from preview.ts

const meta: Meta<typeof MyAtom> = {
  title: 'Components/Current/Atoms/My Atom',
  component: MyAtom,
};

export default meta;
type Story = StoryObj<typeof MyAtom>;

export const Default: Story = {};
```
