# Arda Design System and Style Guide

**Source of Truth:** [Arda MVP 2.0 &#8212; Hi-Fi Mockups](https://www.figma.com/design/JJJ5Yb7t8hMe2LfCX0W5gQ/Arda-MVP-2.0---Hi-Fi-Mockups) (File key: `JJJ5Yb7t8hMe2LfCX0W5gQ`)

This guide defines the visual language, layout patterns, and component conventions for the Arda application. It is intended for both human front-end engineers and AI coding agents building UI in the `ux-prototype` Storybook component library.

## 4. Page Layout

### 4.1 Overall Structure

Every authenticated page follows this layout:

```
+--------------------------------------------------+
| Sidebar (fixed)  |        Content Area            |
| w-[240px] or     | +----------------------------+ |
| w-[56px]         | | Page Header (breadcrumb +   | |
| collapsed        | | title + actions)            | |
|                  | +----------------------------+ |
| Dark bg #0A0A0A  | | Tab Navigation (optional)   | |
|                  | +----------------------------+ |
| - Logo           | |                            | |
| - Nav items      | |  Page Content              | |
| - User footer    | |  (tables, cards, forms...) | |
|                  | |                            | |
+------------------+----------------------------+---+
```

### 4.2 Sidebar

- **Width:** 240px expanded, 56px collapsed
- **Background:** `#0A0A0A` (near-black) with a subtle gradient overlay
- **Position:** Fixed, full viewport height (`fixed inset-y-0 left-0`)
- **Z-index:** 50
- **Transition:** `transition-all duration-200` on width change

**Sidebar sections (top to bottom):**

1. **Logo area** (h-14, border-b border-white/10) &#8212; `ArdaLogoFull` expanded, `ArdaLogo` (icon) collapsed
2. **Navigation** (flex-1, scrollable) &#8212; Icon + label pairs, grouped with optional group labels
3. **User footer** &#8212; Avatar + name + email (hidden when collapsed), logout button

**Nav item states:**

- Default: `text-white/70 hover:bg-white/10`
- Active: `bg-white/10 text-white` with orange left border indicator (`border-l-2 border-[#FC5A29]`)

**Sidebar component sets in Figma:** `Sidebar` / `SidebarMenuButton` / `SidebarGroup` / `SidebarGroupLabel` / `SidebarMenuItem` / `SidebarMenuSub` / `SidebarMenuSubItem` / `CollapseIcon` / `Header Button`

### 4.3 Page Header

Every page has a header component (`Pro Blocks / Page Header / 7.`):

```
+-----------------------------------------------------------+
| Breadcrumb: Home > Section > Current Page                 |
| Page Title                              [Action Buttons]  |
+-----------------------------------------------------------+
```

- Breadcrumb uses `text-sm text-muted-foreground` with `/` or `>` separators
- Page title: `text-xl font-extrabold` or `heading-md`
- Action buttons: Right-aligned, primary + secondary variants

### 4.4 Tab Navigation

Below the page header, many pages use horizontal tab navigation (`Pro Blocks / Tab Nav Link`):

- Tabs use `text-sm font-medium`
- Active tab: Bottom border in primary orange, `font-semibold`
- Inactive: `text-muted-foreground`, hover state

### 4.5 Content Area

- **Left margin:** matches sidebar width (240px or 56px)
- **Padding:** `p-6` (24px) all around
- **Background:** `#ffffff`
- **Max width:** Content is typically fluid within the available space

### 4.6 Side Drawers / Sheets

Used for item detail editing and viewing:

- Slides in from the right
- Width: typically 400&#8211;600px
- Has its own header with close button
- Overlay/backdrop on the rest of the content
- Uses `Component/Side Drawer/Item` and `Sheet` component patterns

### 4.7 Responsive Behavior

- **Desktop-first** design (primary target is desktop browsers)
- Mobile shows a &#8220;device check&#8221; screen encouraging desktop use (`Mobile device check` frame)
- Sidebar collapses to icon-only (56px) mode on smaller viewports
- Mobile scanning mode exists as a separate flow

**Sizes:**

- `sm`: height 32px (2rem), `text-xs`, `px-3`
- `md`: height 36px (2.25rem), `text-sm`, `px-4`
- `lg`: height 40px (2.5rem), `text-sm`, `px-6`

**States:** Default, Hover (slightly darker), Pressed, Disabled (50% opacity), Loading (spinner + disabled)

**Split Button:** Primary action + dropdown chevron for additional options (`.Split button` component set)

**Border radius:** `rounded-md` (8px) for all buttons

### 5.2 Badges

**Component:** `ArdaBadge` (Figma: `Badge` component set)

Six variants matching the status colors: `default` / `success` / `warning` / `info` / `destructive` / `outline`. Optional dot indicator for active/live status. Styled with `text-xs font-medium`, `rounded-sm` (6px), `px-2 py-0.5`.

**Badge Number:** Numeric counter badge (e.g., notification count), small circle, `text-xs font-bold`.

### 5.3 Cards

**Component:** `Card` component set

- Background: White
- Border: 1px `border-border` (`#E5E5E5`)
- Shadow: `shadow-sm`
- Border radius: `rounded-xl` (14px) for large cards, `rounded-lg` (10px) for standard
- Padding: `p-4`
- Optional blue accent bar at top (4px height, `#3B82F6`)

### 5.4 Form Controls

#### Input
**Figma:** `Input` component set. Height: 36px (`h-9`), border: 1px `border-input`, radius: `rounded-md`, font: `text-sm`, placeholder: `text-muted-foreground`. States: Default, Focused, Error (red border + error message), Disabled.

#### Select
**Figma:** `Select` component set. Same dimensions as Input. Chevron down icon on right. Dropdown uses `shadow-md`, `rounded-md`.

#### Combobox
**Figma:** `Combobox` component set. Search input + dropdown list. Filterable options. Used for typeahead/autocomplete patterns.

#### Textarea
**Figma:** `Textarea` component set. Min height: 80px. Same border/radius as Input. Resizable.

#### Switch
**Figma:** `Switch` component set. Toggle between on/off states. Primary orange when active.

#### File Drop Zone
**Figma:** `File Drop Zone` / `File Drop Zone Item` component sets. Dashed border area for drag-and-drop uploads. Shows file items after upload with remove option.

### 5.5 Tables

Tables are a primary data display pattern in Arda (item management, order queue, receiving).

- **Header row:** `bg-gray-50` (`#F9FAFB`), `text-xs font-medium text-muted-foreground uppercase`
- **Body rows:** White background, `text-sm`, `hover:bg-gray-100`
- **Cell padding:** `px-4 py-3`
- **Borders:** Horizontal dividers between rows (`border-b border-border`)
- **Row actions:** Right-aligned column with icon buttons or context menu
- **Sortable columns:** Header with sort icon indicator

### 5.6 Alerts and Toasts

**Alert** (`Alert` component set): Bordered box with icon + title + description. Variants follow status colors. `rounded-lg`, `p-4`.

**Toast** (`Toast / Close Button` component set): Appears at bottom-right or top-right. Auto-dismisses. Close button (X icon). Success/error variants.

### 5.7 Dialogs and Modals

- Centered overlay with backdrop
- `rounded-xl`, `shadow-lg`
- Header: Title + optional description + close button
- Footer: Action buttons (primary right-aligned, cancel left)
- Max width varies: sm (400px), md (500px), lg (640px)

### 6.4 Icon Color

Icons inherit text color. Use the same color tokens as text:

- Interactive: `text-foreground hover:text-[var(--base-primary)]`
- Destructive: `text-[var(--base-destructive)]`

## 7. Screen Patterns

### 7.1 Authentication Screens (Unauthenticated)

Layout: Centered card on full-page background (no sidebar). Screens: Sign In, Account Creation, Password Reset, Confirm Email, Email Verified. Error states use inline field validation (red border + error text below field). Background uses the `Background` component set (gradient variants).

### 7.2 Manage Items (Primary Data Page)

```
+--sidebar--+-------------------------------------+
|           | Breadcrumb: Home > Manage Items     |
|           | Manage Items        [+ Add Item]    |
|           | [All] [Draft] [Recently Imported]   |

|           | | Name | SKU | Status | Actions |  |
|           | |------|-----|--------|---------|  |
|           | | ...  | ... | Badge  | ... ... |  |
+           +-------------------------------------+
```

- **Tab navigation** for filtering views (All / Draft / Recently Imported / etc.)
- **Data table** as primary content
- **Empty state** with illustration (Puddles) and CTA button
- **Side drawer** opens for item details/editing

### 7.3 Order Queue

Items grouped by supplier in cards. Card has supplier info header + item rows. &#8220;Email sent&#8221; toast notification on order completion. Animation for &#8220;add to queue&#8221; action.

### 7.4 Admin / Account Profile

Tab-based sections within the page (Profile / Company / Preferences). Form-heavy layout with grouped fields. Multiple variants for different admin sub-pages.

### 7.5 Scanning Mode

Desktop and mobile variants exist:

- **Desktop:** Sidebar content area with camera feed / scan overlay
- **Mobile:** Full-screen scanning interface (separate flow)

### 7.6 Modals and Dialogs

Used extensively for confirm delete actions, import data flows, form submission confirmations, and item detail popovers. Pattern: Centered, overlaid, with clear primary/secondary action buttons.

## 8. Implementation Conventions

### 8.1 Component Structure

Components follow a **separated concerns** pattern:

```
src/components/[level]/[component-name]/
  [component-name].tsx        # Component implementation
  [component-name].css        # Component-specific styles (optional)
  [component-name].stories.tsx # Storybook stories
  [component-name].test.tsx   # Unit tests
```

**Levels:** `atoms/` &#124; `molecules/` &#124; `organisms/`

### 8.2 Props Pattern

Components use a three-tier interface pattern:

```typescript
/** Design-time configuration - structural, set at composition time */
export interface ArdaComponentStaticConfig {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
}

/** Runtime configuration - changes during component lifetime */
export interface ArdaComponentRuntimeConfig {
  loading?: boolean;
  disabled?: boolean;
}

/** Combined props - extends HTML element attributes */
export interface ArdaComponentProps
  extends ArdaComponentStaticConfig,
    ArdaComponentRuntimeConfig,
    React.HTMLAttributes<HTMLElement> {}
```

### 8.3 Naming Conventions

- **Component names:** `Arda[ComponentName]` prefix (e.g., `ArdaButton`, `ArdaSidebar`)
- **File names:** kebab-case (e.g., `button.tsx`, `item-card.tsx`)
- **CSS classes:** `arda-[component]` prefix (e.g., `arda-btn`, `arda-btn-primary`)
- **Type exports:** `Arda[Component]Props`, `Arda[Component]StaticConfig`, `Arda[Component]RuntimeConfig`

### 8.4 Styling Approach

1. **Prefer Tailwind utility classes** for layout and spacing
2. **Use CSS custom properties** (`var(--base-primary)`) for design tokens
3. **Use `cn()` utility** for conditional class composition
4. **Component-specific CSS files** (`.css`) for complex component styles, using `arda-` prefixed class names
5. **Never use inline `style` objects** except for truly dynamic values (e.g., computed gradients)

### 8.5 Common Tailwind Patterns

```tsx

<div className="rounded-xl border border-border bg-white p-4 shadow-sm">

// Section heading
<h2 className="text-lg font-semibold text-foreground">

// Primary button (when not using ArdaButton)

// Table header cell
```
