# Applications

This page documents an older Storybook pattern that predated the current `App` and `Prototypes` organization. It is retained in `Archive` for historical reference.

## Overview

The original application-mock approach used three variants:

This approach has been superseded by:

- `App/Current` for vendored production-parity pages
- `App/Migration` for page-level migration work
- `Prototypes` for forward-looking workflow concepts

## How to Add a Page

Do not use this pattern for new work. Prefer `App/Current`, `App/Migration`, or `Prototypes` depending on intent.

Historically, teams created a directory like this:

```
src/applications/my-page/
  MyPage.tsx
  MyPage.stories.tsx
```

The page component itself followed this pattern:

```tsx
import { Button } from '@/components/extras/atoms/Button/Button';
import { Card } from '@/components/extras/molecules/Card/Card';

export function MyPage() {
  return (
    <div className="mx-auto max-w-4xl space-y-6 p-8">
      <h1 className="text-2xl font-bold">My Page</h1>
      <Card>
        <p>Page content goes here.</p>
        <Button>Take Action</Button>
      </Card>
    </div>
  );
}
```

And the story usually looked like:

```tsx
import type { Meta, StoryObj } from '@storybook/react-vite';
import { MyPage } from './MyPage';

const meta = {
  title: 'Archive/Applications/My Page',
  component: MyPage,
  parameters: {
    layout: 'fullscreen',
  },
} satisfies Meta<typeof MyPage>;

export default meta;
type Story = StoryObj<typeof meta>;

export const Default: Story = {};
```

## Using Custom Components

Sometimes a page mock needed a component that did not yet exist in the library. Teams defined it locally within the mock directory:

```
src/applications/my-page/
  MyPage.tsx
  MyPage.stories.tsx
  components/
    CustomWidget.tsx    # Local to this mock
```

Import it in the page:

```tsx
import { CustomWidget } from './components/CustomWidget';
```

That kept the library clean while still allowing rapid prototyping.

## Promoting to Library

When a custom component proves useful across multiple mocks, promote it to the library:

1. Move the file to the appropriate level: `src/components/atoms/`, `molecules/`, or `organisms/`.
2. Create a standalone story file.
3. Add unit tests.
4. Export it from `src/index.ts`.
5. Update all imports in the application mocks that used the local version.
