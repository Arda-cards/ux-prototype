# Arda User Experience

This Storybook site is the living documentation and prototype gallery for **@arda-cards/design-system** &#8212; the shared React component library for the Arda platform.

It serves three audiences:

- **Designers** review visual fidelity, interaction patterns, and full-page application mocks.
- **Developers** discover reusable components, read API documentation, and run interaction tests.
- **Product owners** walk through end-to-end use cases and validate acceptance criteria.

## How the Sidebar Is Organized

The sidebar groups content into seven top-level sections:

### Start Here

Orientation pages for new contributors:

- Overview
- Getting Started
- Storybook changelog

### Guides

Deep documentation for implementation, workflows, design, tooling, and reference material.

### Foundations

Visual tokens and shared design primitives:

- Colors
- Brand Assets
- Icons

### Components

Reusable UI library stories grouped by track:

- **Current** &#8212; the default track for reusable components in active use.
- **Migration** &#8212; extraction and replacement work that is not yet promoted.

Both tracks are organized by atomic level (Atoms, Molecules, Organisms).

### App

Page-level application views grouped for parity and migration work:

- **Current** &#8212; vendored parity pages from `arda-frontend-app`.
- **Migration** &#8212; refactor variants of those same pages.

### Prototypes

Forward-looking workflow prototypes used to validate product direction before production integration.

### Archive

Historical mocks retained for reference only.

## Component Architecture

All components follow the **interface separation pattern** documented in the Component Guidelines page. In brief:

- **StaticConfig** &#8212; properties chosen at design time that never change (labels, variants, structure).
- **RuntimeConfig** &#8212; properties driven by application state that change during the component lifetime (loading, selection, event handlers).
- **InitConfig** &#8212; optional mount-time properties for one-time setup (async data fetching).

Component names use the `Arda` prefix (e.g., `ArdaButton`, `ArdaDataGrid`, `ArdaItemsDataGrid`). Files use kebab-case. Directories are organized by atomic level.
