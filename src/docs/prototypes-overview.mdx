# Prototypes Section Overview

`Prototypes` contains forward-looking workflow scenarios and conceptual flows.

These stories are not intended to be strict production mirrors. They are used to:

- Validate UX direction for upcoming capabilities
- Iterate on flows before production implementation
- Communicate product intent with concrete interactive artifacts

If you need parity with the current app, use `App/Current` instead.
