# Atoms

Atoms are the smallest, most fundamental UI building blocks. They are standalone components that do not depend on other custom components and can be used anywhere in the application.

## Criteria

- Contains no domain logic or business rules
- Does not import other Arda atoms, molecules, or organisms
- Highly reusable across different contexts
- Generally maps to a single HTML element or a thin wrapper around a library primitive
- Focuses on presentation and basic interaction patterns

## Subcategories

Atoms are organized into lightweight groups based on purpose. The group determines the Storybook sidebar path.

## File structure

Each atom lives in its own directory under `src/components/atoms/`:

```
atoms/
  badge/
    badge.tsx            # Component implementation
    badge.stories.tsx    # Stories
    badge.mdx            # Guides (optional, uses Meta of={Stories})
    badge.test.tsx       # Tests (if applicable)
  button/
    ...
```

## Adding a new atom

1. **Create the directory** at `src/components/atoms/{component-name}/`.
2. **Implement the component** in `{component-name}.tsx`.
3. **Choose the group**:
   - top-level atom (`Components/Stable/Atoms/{ComponentName}`)
   - Form atom (`Components/Stable/Atoms/Form/{ComponentName}`)
   - Grid atom (`Components/Stable/Atoms/Grid/{ComponentName}`)
4. **Create stories** in `{component-name}.stories.tsx` with the matching title path.
5. **Optionally create docs** in `{component-name}.mdx` using `<Meta of={Stories} />` to attach to the story.
6. **Export from** `src/index.ts` if the component is part of the public API.

### Story title example

```tsx
const meta: Meta<typeof MyComponent> = {
  title: 'Components/Stable/Atoms/Form/MyComponent',
  component: MyComponent,
};
```

### Deciding the group

- Does it handle user input? &#8594; **Form**
- Is it an AG Grid cell editor or renderer? &#8594; **Grid**
- Otherwise, keep it at the atom root (**Core**)
