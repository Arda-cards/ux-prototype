# Canary Primitives

Stock shadcn/ui components installed unchanged into the Arda component library.

**Layer**: Primitives &#8212; peer of `atoms/`, the lowest level in the canary hierarchy.

---

## Purpose

Primitives are raw shadcn/ui building blocks. They are:

- **Installed as-is** &#8212; no Arda-specific customization beyond import path.
- **Internal only** &#8212; not exported from `canary.ts`. Import primitives directly.
- **Regenerated via CLI** &#8212; do not hand-edit primitive source files.

When a primitive needs customization (e.g., a badge with count logic), promote it
to an atom in `atoms/` instead of modifying the primitive.

## Import convention

```tsx
import { DropdownMenu, DropdownMenuContent, DropdownMenuItem, DropdownMenuTrigger }
  from '@/components/canary/primitives/dropdown-menu';

import { SidebarProvider, SidebarInset, SidebarTrigger }
  from '@/components/canary/primitives/sidebar';

import { Tabs, TabsList, TabsTrigger, TabsContent }
  from '@/components/canary/primitives/tabs';
```

Import from the individual primitive file, not from a barrel. Each primitive file
corresponds to a single shadcn component.

---

## Regenerating a primitive

To update or add a primitive, run the shadcn CLI in `ux-prototype/`:

```sh
npx shadcn@latest add <component-name>
```

This overwrites the file in `src/components/canary/primitives/`. Review the diff
before committing &#8212; shadcn may change import paths or Tailwind classes.

---

## Primitives vs atoms

| | Primitives | Atoms |
|---|---|---|
| Source | Stock shadcn &#8212; CLI generated | Custom &#8212; hand-crafted |
| Exported | No | Yes (via `canary.ts`) |
| Modified | No &#8212; regenerate via CLI | Yes &#8212; hand-edited |
| Stories | Integration check only | Full story + test + MDX |

When a use case requires a modified version of a primitive (e.g., Badge with count capping),
create an atom in `atoms/` that wraps the primitive &#8212; do not fork the primitive itself.
