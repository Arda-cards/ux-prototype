# ImageDropZone

Unified image input surface for the Arda design system. Accepts images via drag-and-drop,
file picker, or URL entry &#8212; all from a single component.

**Molecule** &#8212; Canary | Import: `@/components/canary/molecules/image-drop-zone`

---

## Input Methods

`ImageDropZone` classifies each submission as one of three `ImageInput` union variants:

<table>
  <thead>
    <tr>
      <th>Method</th>
      <th>Resulting type</th>
      <th>Validation</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Drag and drop a file</td>
      <td><code>&#123; type: 'file', file &#125;</code></td>
      <td>MIME type must be in <code>acceptedFormats</code></td>
    </tr>
    <tr>
      <td>Upload from computer (file picker)</td>
      <td><code>&#123; type: 'file', file &#125;</code></td>
      <td>MIME type must be in <code>acceptedFormats</code></td>
    </tr>
    <tr>
      <td>Paste image URL &#8594; Enter</td>
      <td><code>&#123; type: 'url', url &#125;</code></td>
      <td>Must begin with <code>https://</code></td>
    </tr>
  </tbody>
</table>

On validation failure the component emits `{ type: 'error', message }` and shows an
inline error message below the drop zone.

---

## Error Handling

Errors are displayed inline in `text-destructive` beneath the input area. Two classes of errors
are handled:

- **Invalid file type** &#8212; the dropped or selected file's MIME type is not in `acceptedFormats`.
- **Non-HTTPS URL** &#8212; the submitted URL does not start with `https://`.
