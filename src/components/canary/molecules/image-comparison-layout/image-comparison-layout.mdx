# ImageComparisonLayout

Responsive molecule for comparing an existing image against a new one. On desktop the two
panels sit side-by-side; on mobile they collapse into a two-tab interface.

**Molecule** &#8212; Canary | Import: `@/components/canary/molecules/image-comparison-layout`

---

## Overview

`ImageComparisonLayout` wraps the left "Current" panel (an `ImageDisplay`) and the right "New"
panel (caller-supplied `children`, typically an `ImagePreviewEditor`). The layout responds to
the viewport:

- **Desktop (md: and above)** &#8212; flex-row. Left panel is a fixed-width reference thumbnail
  with a `border-r` divider. Right panel expands to fill the remaining space.
- **Mobile (below md:)** &#8212; `Tabs` component with "Current" and "New" triggers. Defaults to
  the "New" tab so the user immediately sees the upload area.

When `existingImageUrl` is `null` the component passes through directly to `children` with no
comparison chrome &#8212; useful for new-record forms where no existing image exists yet.

## No Existing Image

When `existingImageUrl` is `null` no comparison UI is shown &#8212; only the `children` are
rendered. This is the pass-through mode for new-record forms.
