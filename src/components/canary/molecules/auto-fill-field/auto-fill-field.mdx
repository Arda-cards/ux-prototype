# AutoFillField

Wraps any form field to show an auto-fill badge indicator. On hover, the sparkle badge expands to reveal the source name (e.g. "Filled by Amazon"). Takes no extra vertical space &#8212; the badge is absolutely positioned over the field's top-right corner.

**Molecule** &#8212; Canary | Import: `import { AutoFillField } from '@arda-cards/design-system/canary'`

---

## Dismiss Modes

AutoFillField supports three dismiss modes to handle different input types:

  <thead>
    <tr>

  </thead>
  <tbody>
    <tr>
      <td><code>input</code> (default)</td>
      <td><code>onInput</code></td>
      <td>Text inputs, textareas, typeaheads &#8212; fires on every keystroke</td>
    </tr>
    <tr>
      <td><code>change</code></td>

      <td>Native selects, checkboxes, toggles, file inputs</td>
    </tr>
    <tr>
      <td><code>manual</code></td>
      <td>None</td>
      <td>Radix Select, image upload, any custom component that doesn&#8217;t fire native DOM events</td>
    </tr>
  </tbody>
</table>

## Usage

### Text input (default &#8212; dismissOn="input")

```tsx
<AutoFillField source="Amazon" onClear={() => clearAutoFill('sku')}>
  <InputGroup>
    <InputGroupInput value={sku} onChange={handleChange} />
  </InputGroup>
</AutoFillField>
```

The badge disappears when the user types in the input.

### Native select (dismissOn="change")

```tsx
<AutoFillField source="Amazon" dismissOn="change" onClear={() => clearAutoFill('taxable')}>
  <Switch checked={taxable} onCheckedChange={handleChange} />
</AutoFillField>
```

The badge disappears when the native element fires a `change` event.

### Custom component (dismissOn="manual")

```tsx
<AutoFillField source="Amazon" dismissOn="manual">
  <ArdaSelect
    onValueChange={(v) => {
      setValue(v);
      clearAutoFill('orderMethod'); // one extra line
    }}
  />
</AutoFillField>
```

Custom components like Radix Select and image dropzones use React callbacks instead of native DOM events. These don&#8217;t bubble to a wrapper div, so the consumer clears the source from their own callback.

## Badge Behavior

- **Collapsed:** Small sparkle icon in a pill at the top-right corner
- **Hover:** Expands smoothly to show "Filled by &#123;source&#125;"
- **On dismiss:** Badge disappears (source becomes undefined)
- **No source:** Wrapper renders transparently &#8212; zero overhead

---

## Accessibility

- The badge is decorative (`select-none`, `cursor-default`) &#8212; not interactive
- The wrapper uses event delegation to detect edits without modifying child components
- Screen readers ignore the badge (no accessible label)
