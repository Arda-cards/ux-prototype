# AutoFillField

Wraps any form field to show an auto-fill badge indicator. On hover, the sparkle badge expands to reveal the source name (e.g. "Filled by Amazon"). Takes no extra vertical space &#8212; the badge is absolutely positioned over the field's top-right corner.

**Molecule** &#8212; Canary | Import: `import { AutoFillField } from '@arda-cards/design-system/canary'`

## Dismiss Modes

AutoFillField supports three dismiss modes to handle different input types:

<table>
  <thead>
    <tr>
      <th>Mode</th>
      <th>DOM Event</th>
      <th>Use for</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <code>input</code> (default)
      </td>
      <td>
        <code>onInput</code>
      </td>
      <td>Text inputs, textareas, typeaheads &#8212; fires on every keystroke</td>
    </tr>
    <tr>
      <td>
        <code>change</code>
      </td>
      <td>
        <code>onChange</code>
      </td>
      <td>Native selects, checkboxes, toggles, file inputs</td>
    </tr>
    <tr>
      <td>
        <code>manual</code>
      </td>
      <td>None</td>
      <td>
        Radix Select, image upload, any custom component that doesn&#8217;t fire native DOM events
      </td>
    </tr>
  </tbody>
</table>

### Why `manual`?

Custom components like Radix Select and image dropzones use React callbacks (`onValueChange`, `onInput`) instead of native DOM events. These callbacks don&#8217;t bubble to a wrapper `div`, so the wrapper can&#8217;t detect them automatically.

In manual mode, the consumer clears the `source` from their own callback &#8212; just one extra line of code:

```tsx
<AutoFillField source="Amazon" dismissOn="manual">
  <ArdaSelect
    onValueChange={(v) => {
      setValue(v);
      clearAutoFill('orderMethod'); // one extra line
    }}
  />
</AutoFillField>
```

## Badge Behavior

- **Collapsed:** Small sparkle icon in a pill at the top-right corner of the wrapped field
- **Hover:** Expands smoothly to show "Filled by {source}"
- **On dismiss:** Badge disappears (source becomes undefined)
- **No source:** Wrapper renders transparently with no badge &#8212; zero overhead

---

## Accessibility

- The badge is `select-none` and `cursor-default` &#8212; it is not interactive
- The wrapper uses `onInput` or `onChange` event delegation to detect edits without modifying child components
- The sparkle icon has no accessible label &#8212; the badge is decorative, not actionable
