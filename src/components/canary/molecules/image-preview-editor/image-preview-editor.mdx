# ImagePreviewEditor

Crop and edit an image at a locked aspect ratio. Provides zoom, rotation, and reset controls
via a compact toolbar beneath the crop area.

**Molecule** &#8212; Canary | Import: `@/components/canary/molecules/image-preview-editor`

---

`onCropComplete`, `onZoomChange`, and `onRotationChange` call.

## Baked-In Behaviors

### Crop Area

The crop viewport is rendered by `react-easy-crop` and fills the available container height. The `aspectRatio` prop is locked &#8212; the crop selection box cannot be resized by the user. Panning is performed by dragging inside the crop area; the image moves under the fixed selection window.

### Zoom Slider

The zoom slider has a range of 0.5&#215; (minimum, zoomed out) to 3&#215; (maximum, zoomed in) with a step of 0.1. The initial zoom value on mount and after reset is 1&#215;, which positions the slider in the lower portion of its range. This allows users to zoom both in and out from the default view.

Every slider change fires `onZoomChange` synchronously with the updated zoom value.

### Rotation Buttons

Two rotation buttons sit in the toolbar row below the zoom slider:

- **Clockwise** button: adds +90&#176; to the current rotation value, wrapping from 270&#176; back to 0&#176;.
- **Counter-clockwise** button (rendered with the Undo2 icon): subtracts 90&#176; from the current rotation, wrapping from 0&#176; back to 270&#176;.

Each button press fires `onRotationChange` with the updated rotation.

### Reset

The Reset button restores all three editable dimensions to their initial values:

- Crop position: centered on the image
- Zoom: 1&#215;
- Rotation: 0&#176;

After resetting internal state, the component calls `onReset()` to notify the parent. This allows the parent (e.g., `ImageUploadDialog`) to clear any previously stored `CropData` and treat the image as unedited.

### File and Blob Handling

When `imageData` is a `File` or `Blob`, the component calls `URL.createObjectURL()` on mount and stores the resulting object URL. This URL is passed to `react-easy-crop` as the image source. On unmount, the component calls `URL.revokeObjectURL()` to release the browser memory. When `imageData` is already a `string` (URL or data URL), it is used directly with no object URL lifecycle management.

### Toolbar Layout

The toolbar below the crop area is arranged in two rows:

1. **Top row**: the zoom slider spanning the full width, labeled with zoom-out and zoom-in icons on either end.
2. **Bottom row**: the counter-clockwise rotation button on the left, the clockwise rotation button in the center, and the Reset button on the right.

### Event Emission

Three independent callbacks fire as the user interacts with the editor:
`onCropComplete` is called when the user finishes resizing/repositioning the crop area,
`onZoomChange` fires on every zoom slider movement, and `onRotationChange` fires on
each rotate-CW or rotate-CCW button click. Each callback receives only the value that
changed.

## Reset

The Reset button restores all edit state to defaults and calls `onReset`.
Use this to let users abandon edits and return to the original image.
