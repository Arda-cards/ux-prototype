## Extending columns

The organism ships with 8 curated columns. Consumers needing the full 40+ columns from arda-frontend can import and extend:

```tsx
import { itemGridColumnDefs } from '@arda-cards/design-system/canary';

const extendedColumns = [
  ...itemGridColumnDefs,
  { headerName: 'GL Code', field: 'generalLedgerCode', width: 140 },
  { headerName: 'Facility', valueGetter: (p) => p.data?.locator?.facility },
];
```

## Adding editing

The organism is read-only by default. For cell editing, use `ArdaDataGrid` directly with `enhanceEditableColumnDefs` from the extras presets &#8212; the organism doesn't need to know about editing.

## Consumer responsibilities

- **Pagination** &#8212; The organism is client-side only. For server-side pagination, compose `ArdaDataGrid` directly with `paginationData` props.
- **Draft management** &#8212; App-level concern. Use `onCellValueChanged` on `ArdaDataGrid` with a custom dirty-tracking hook.
- **Bulk actions** &#8212; Wire `onSelectionChange` to your own toolbar (see Composition story).
