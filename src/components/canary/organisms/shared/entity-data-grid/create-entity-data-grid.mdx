# Entity Data Grid

General-purpose data grid factory for building typed, feature-rich entity list views. All entity grids in Arda (ItemGrid, supplier grid, etc.) are built on this foundation via the `createEntityDataGrid<T>(config)` factory function.

**Organism** &#8212; Canary | Factory: `createEntityDataGrid<T>(config)` | Import: `@arda-cards/design-system/canary`

## Purpose

`createEntityDataGrid` is the single entry point for building any entity list view. It encapsulates:

- **Row-auto-publish editing** &#8212; edit cells, blur the row, changes auto-save with visual feedback
- **Client-side search** &#8212; debounced text filter across configured fields with count display
- **Actions column** &#8212; pinned-right column for row-level actions (view, edit, delete)
- **Pagination** &#8212; server-driven or client-side (mutually exclusive, chosen at design time)
- **Toolbar slot** &#8212; custom ReactNode rendered alongside the search bar
- **Drag-to-scroll** &#8212; horizontal mouse drag on wide grids
- **Auto-height** &#8212; grid grows to fit content
- **Arda theming** &#8212; `themeQuartz` with design-system token mapping

Entity-specific grids like `ItemGrid` configure these capabilities via the factory and add domain-specific columns, typeahead editors, and custom renderers.

## Editing Lifecycle

The entity-data-grid uses **row-auto-publish** &#8212; the only editing mode:

1. User double-clicks a cell &#8594; AG Grid enters edit mode
2. User changes the value &#8594; change recorded in pending changes map
3. User edits more cells in the same row &#8594; changes accumulate
4. User leaves the row (clicks another row, tabs out) &#8594; 50ms debounce
5. Row enters &#8220;saving&#8221; state (light orange background via <code>.ag-row-saving</code>)
6. <code>onRowPublish(rowId, changes, entity?)</code> fires asynchronously
7. On success &#8594; row returns to idle, pending changes cleared
8. On failure &#8594; row enters &#8220;error&#8221; state (light red via <code>.ag-row-error</code>)

## Usage

```tsx
import { createEntityDataGrid } from '@arda-cards/design-system/canary';
import type { Item } from '@/types';

const { Component: ItemGrid } = createEntityDataGrid<Item>({
  displayName: 'ItemGrid',
  persistenceKeyPrefix: 'item-grid',
  columnDefs: itemGridColumnDefs,
  defaultColDef: itemGridDefaultColDef,
  getEntityId: (item) => item.entityId,
  searchConfig: { fields: ['name', 'internalSKU'], placeholder: 'Search items...' },
  enableDragToScroll: true,
  paginationMode: 'client',
  pageSize: 25,
});

// Use in a component:
function ItemsPage() {
  const [items, setItems] = useState<Item[]>([]);
  const gridRef = useRef<EntityDataGridRef>(null);

  return (
    <ItemGrid
      ref={gridRef}
      data={items}
      loading={isLoading}
      enableCellEditing
      onRowPublish={async (rowId, changes) => {
        await api.updateItem(rowId, changes);
      }}
      onRowClick={(item) => openDetailPanel(item)}
      toolbar={<Button onClick={() => gridRef.current?.saveAll()}>Save All</Button>}
    />
  );
}
```
