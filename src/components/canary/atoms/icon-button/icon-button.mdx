# IconButton

Icon-only button with required accessible label, optional notification badge, and auto-shown tooltip.

## Purpose

`IconButton` wraps `Button` (ghost variant, `icon` size) with three Arda-specific additions:

- A **required `label`** prop that populates `aria-label` and, by default, appears as a bottom-side tooltip on hover.
- An optional **`badgeCount`** for notification-style counts. Counts above 99 display as `99+`.
- A **`showTooltip`** flag (default `true`) to suppress the tooltip when context already makes the action clear.

Used in `AppHeader` for icon actions such as Help, Notifications, and the user menu.

## Accessibility

- `label` is required and maps directly to `aria-label`. Screen readers announce the button's purpose.
- The notification badge carries its own `role="status"` `aria-label` (e.g., `"8 notifications"`).
- `IconButton` always renders as a ghost button; it does not expose a `variant` prop &#8212; use `Button` directly for other variants.
- Wrap `IconButton` (or its parent) in a `TooltipProvider` when rendering outside the default app layout.
