## Purpose

`Drawer` slides in from the edge of the viewport (right by default). Use it for detail panels, edit forms, and contextual actions that need more space than a modal dialog. For simple confirmations, use `Dialog` instead.

The drawer is controlled: the caller owns the `open` state and provides `onOpenChange` to handle close events (close button, overlay click, Escape key). Auto-focus is suppressed on open so focus management can be handled explicitly by the consuming component.

## Usage pattern

```tsx
function ItemDetailPanel({ itemId, onClose }: Props) {
  const [open, setOpen] = React.useState(true);

  return (
    <Drawer open={open} onOpenChange={(v) => { setOpen(v); if (!v) onClose(); }}>
      <DrawerHeader>
        <DrawerTitle>Item Details</DrawerTitle>
        <DrawerDescription className="sr-only">View and manage this inventory item.</DrawerDescription>
      </DrawerHeader>
      <DrawerBody className="px-6 py-4 space-y-3">
        <ReadOnlyField label="SKU" value="WDG-4420-BLK" />
        <ReadOnlyField label="Supplier" value="McMaster-Carr" />
      </DrawerBody>
      <DrawerFooter>
        <Button variant="outline" onClick={() => setOpen(false)}>Close</Button>
      </DrawerFooter>
    </Drawer>
  );
}
```

## Accessibility

- `DrawerTitle` is required &#8212; it provides the accessible name for the dialog role. Use `className="sr-only"` to hide it visually while keeping it available to screen readers.
- `DrawerDescription` is optional but recommended for complex drawers.
- Focus is not auto-moved on open (auto-focus is suppressed); set focus explicitly on the first interactive element if needed.
- Pressing Escape or clicking the backdrop closes the drawer and calls `onOpenChange(false)`.
