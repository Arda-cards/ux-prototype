# Button

Canonical action button for all Arda UI. Built-in loading states, tooltip support, `motion-reduce` respect, and consistent focus rings mapped to Arda design tokens.

## Loading State

The `loading` prop accepts a **boolean** or a **string**:

- `loading={true}` &#8212; shows a spinner alongside the existing children text.
- `loading="Downloading&#8230;"` &#8212; shows a spinner and replaces the children with the given text.

Use `loadingPosition="end"` to place the spinner after the text instead of before it.

For icon-only sizes, the spinner replaces the icon entirely regardless of position.

## Tooltip

The `tooltip` prop wraps the button in a Radix Tooltip automatically. When the button is `disabled`, pointer events are handled internally so the tooltip still appears on hover.

Use `tooltipSide` to control placement (`'top'` | `'bottom'` | `'left'` | `'right'`, defaults to `'top'`).

Requires `TooltipProvider` in the app root (already present in Arda layouts).

## Composition: asChild

Sometimes you need something that *looks* like a button but *behaves* like a different element &#8212; most commonly a link. Instead of wrapping a link in a button (which is invalid HTML and breaks accessibility), `asChild` tells Button to transfer all of its styling and props onto its single child element.

The `Button` component itself doesn't render a `<button>` tag. Instead, the child element becomes the rendered output, inheriting all variant classes, focus rings, and size styles.

```tsx
{/* Renders an <a> tag that looks exactly like a ghost button */}
<Button asChild variant="ghost">
  <a href="/items">View all items</a>
</Button>

{/* Works with Next.js Link */}
<Button asChild variant="primary" size="lg">
  <Link href="/onboarding">Get started</Link>
</Button>
```

**When to use:** Navigation that should look like a button. Without `asChild`, you'd need to duplicate button styles on the link or nest a `<button>` inside an `<a>` (which is invalid HTML).

## Accessibility

- Focus ring: 2px `ring` token, visible on keyboard focus via `focus-visible`.
- Destructive variant uses `ring-destructive` for its focus indicator.
- `motion-reduce`: loading spinner animation stops for users who prefer reduced motion.
- Icon-only buttons require `aria-label`.
- Tooltip on disabled buttons works automatically &#8212; no extra wrapper needed.
