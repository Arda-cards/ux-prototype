## Purpose

`Dialog` is built on Radix Dialog primitives with Arda styling (rounded corners, backdrop overlay, design-system spacing). The content panel is portalled to the document body and centered in the viewport.

Use `Dialog` for:
- Confirmation dialogs (delete, discard changes)
- Quick-add or short edit forms
- Error or informational alerts

For slide-over panels with more space, use `Drawer` instead.

## Usage pattern

```tsx
// Controlled dialog
function DeleteConfirmDialog({ open, onClose, onConfirm }: Props) {
  return (
    <Dialog open={open} onOpenChange={(v) => { if (!v) onClose(); }}>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Delete item?</DialogTitle>
          <DialogDescription>This action cannot be undone.</DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <DialogClose asChild>
            <Button variant="outline">Cancel</Button>
          </DialogClose>
          <Button variant="destructive" onClick={onConfirm}>Delete</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}

// Uncontrolled dialog (trigger-based)
function AddItemDialog() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>Add item</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Add new item</DialogTitle>
        </DialogHeader>
        {/* form content */}
      </DialogContent>
    </Dialog>
  );
}
```

## Accessibility

- `DialogTitle` is required &#8212; it provides the accessible name for the `dialog` ARIA role. Radix will warn if it is absent.
- `DialogDescription` is recommended for dialogs with non-obvious purpose.
- Focus is trapped inside the dialog while open. On close, focus returns to the trigger element.
- Pressing Escape dismisses the dialog.
