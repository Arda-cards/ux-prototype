# ArdaDetailField

A read-only label/value pair for entity detail views. Displays a field label above a formatted value with consistent typography using design tokens.

**Classification:** Atom &#8212; Display and Layout

**Track:** Canary

---

## When to Use

Use `ArdaDetailField` whenever you need to display a static attribute of an entity, such as in a detail panel, drawer, or summary card. It is not suitable for editable fields &#8212; use a form input component for those.

Typical uses:
- Item detail panels (SKU, GL Code, Unit Price)
- Supplier or order summary sections
- Read-only profile fields

---

## Configuration Lifecycle

The component uses the **StaticConfig / RuntimeConfig** split:

**StaticConfig** &#8212; chosen at design time, does not change after composition:

- `label` &#8212; The field label shown above the value. Required.
- `fallback` &#8212; Text displayed when `value` is `undefined` or empty. Defaults to an em dash (&#8212;).
- `variant` &#8212; Visual density. Either `default` (standard gap) or `compact` (reduced gap). Defaults to `default`.

**RuntimeConfig** &#8212; driven by application state, may change during the component lifetime:

- `value` &#8212; Plain-text value to display. When `undefined` or empty, the `fallback` is shown.
- `children` &#8212; Custom value rendering (overrides both `value` and `fallback`). Use this for links, badges, or other non-text content.

## Accessibility

`ArdaDetailField` is a purely presentational, read-only component. It renders two `<span>` elements inside a `<div>`. No interactive ARIA roles are applied.

- The label text is visually associated with the value by DOM proximity (label above, value below).
- Both label and value are readable by screen readers in order.
- No keyboard interaction is required for this component.
