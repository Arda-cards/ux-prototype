# Grid Cell Atoms

Specialized cell renderers and editors for use inside AG Grid columns. Each atom provides a display component (read-only rendering) and an editor component (inline editing) for a specific data type.

These atoms depend on AG Grid's component lifecycle (`ICellRendererParams`, `ICellEditorParams`) and should not be used outside a grid context. They are registered in column definitions via the `cellRenderer` and `cellEditor` properties.

**Import**: `import { TextCellDisplay, createTextCellEditor } from '@arda-cards/design-system/canary'`

## Factory Pattern

All editors follow the factory pattern for AG Grid column definition registration:

```tsx
import { createTextCellEditor, TextCellDisplay } from '@arda-cards/design-system/canary';

const columnDefs = [
  {
    field: 'name',
    cellRenderer: (params) => <TextCellDisplay value={params.value} />,
    cellEditor: createTextCellEditor({ maxLength: 100 }),
    editable: true,
  },
];
```

The factory function bakes static configuration into a component that AG Grid instantiates per cell. This avoids re-creating config on every render.

---

## Integration with Entity Data Grid

When using `createEntityDataGrid`, cell atoms are referenced in column definitions passed to the factory config. The entity-data-grid handles the editing lifecycle (auto-publish on row blur) &#8212; cell atoms only need to render and capture values.
