# Image Cell

AG Grid cell renderer and editor for image columns. The display atom wraps
`ImageDisplay` inside `ImageHoverPreview` to provide a 256&#215;256 popover on hover.
Double-click opens `ImageUploadDialog` for uploading or changing the image.

**Grid Atom** &#8212; Display + Editor pair pattern.

---

## Baked-In Behaviors

### ImageCellDisplay

`ImageCellDisplay` is a pure cell renderer. It renders a 28&#215;28px thumbnail (`w-7 h-7`) by composing `ImageHoverPreview` (outer, for the hover popover) and `ImageDisplay` (inner, read-only). The `ImageDisplay` inside the cell renderer is always instantiated without `onImageChange` or `config` &#8212; it is a passive display only. No action icons (eye, trash, pencil) appear inside the cell.

Action icons are deliberately absent from the cell renderer. Their presence would intercept mouse events and prevent AG Grid's native double-click cell editing from triggering. Removing them ensures that a double-click on any part of the cell reaches the grid and opens the cell editor.

The hover popover (256&#215;256px) is provided by the outer `ImageHoverPreview` wrapper. The 500ms hover delay and null suppression behaviors described in the `ImageHoverPreview` documentation apply here without modification.

### ImageCellEditor

`ImageCellEditor` is an AG Grid popup cell editor. It implements the AG Grid cell editor interface:

- `isPopup()` returns `true`, instructing AG Grid to render the editor as a floating popup rather than replacing the cell content in place.
- On mount, the component immediately opens `ImageUploadDialog` so the user sees the dialog without any intermediate state.
- `getValue()` returns the pending new image URL after a confirm, or the original `value` prop if the user cancelled.
- When the user confirms or cancels, `stopEditing()` is called via `setTimeout` (zero delay). The timeout defers the AG Grid call until React has flushed the state update that sets the internal result value. Without this deferral, AG Grid would call `getValue()` before the state update settles, returning a stale value.

### Factory: createImageCellEditor

`createImageCellEditor(config)` is a factory function that curries `ImageFieldConfig` and returns an `ImageCellEditor` component class. This pattern allows the `ImageFieldConfig` (accepted formats, aspect ratio, display names, etc.) to be bound once at column-definition time rather than injected on each cell render:

```tsx
columnDefs: [
  {
    field: 'imageUrl',
    cellRenderer: ImageCellDisplay,
    cellRendererParams: { config },
    cellEditor: createImageCellEditor(config),
    editable: true,
  }
]
```

The returned component is a valid AG Grid cell editor component that can be passed directly to `cellEditor` in a column definition.

## Interactions

### Hover Preview

After hovering over an image cell for ~500 ms, a 256&#215;256 popover appears showing
the full `ImageDisplay`. Moving the pointer away closes the popover immediately.

<Canvas of={Stories.HoverPreview} />

### Double-Click to Edit

Double-click an image cell to open the `ImageUploadDialog`. The dialog shows
the current image for comparison. Confirm commits a new URL; Cancel discards.

The Name column is also editable &#8212; double-click it to verify AG Grid
editing is active.

<Canvas of={Stories.DoubleClickEdit} />

### Keyboard Edit

Tab to a cell in the Image column and press **Enter** to trigger the cell editor.

<Canvas of={Stories.KeyboardEdit} />
