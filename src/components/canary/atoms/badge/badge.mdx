# Badge

Small inline status badge with count-capping, icon support, and collapsible expand-on-hover behavior.

**Atom** &#8212; Canary | Import: `import { Badge } from '@arda-cards/design-system/canary'`

## Badge vs TokenChip

A Badge **annotates** something the user reads: a status, a category, a count,
a role. It describes data owned elsewhere; at most it is dismissible
(`onDismiss`) when it mirrors a filter or tag applied somewhere else.

A [TokenChip](?path=/docs/components-canary-atoms-tokenchip--docs) **is** a
value the user placed into a field and can take back out — a recipient
address, a tag being composed. It always lives inside an editing surface
(recipient rows, `MultiSelectTypeaheadInput`), carries a × remove button, and
can host an inline action (e.g. "set as default").

Rule of thumb: if removing it changes *what you're looking at*, it's a Badge
being dismissed; if removing it changes *what you're about to submit*, it's a
TokenChip.

## Modes

`Badge` supports four rendering modes:

1. **Label** &#8212; pass text via `children` (e.g. `New`, `Draft`).
2. **Count** &#8212; pass `count={n}` for a numeric badge. Capped at `max` (default 99). Uses monospaced tabular-nums. Gets `role="status"` for accessibility.
3. **Icon** &#8212; pass `icon={Component}`. Without children, renders as a compact circle. With children, renders icon + text.
4. **Collapsible** &#8212; pass `icon` + `collapsible`. Shows icon-only at rest (circle), expands on hover to reveal text (pill). Fixed height in both states.

## Accessibility

- Count mode: receives `role="status"` for screen reader live updates.
- Icon-only: decorative &#8212; use `aria-hidden="true"` if no accessible meaning.
- Collapsible: expanded text provides additional context on hover.
