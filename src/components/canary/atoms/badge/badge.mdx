# Badge

Small inline status badge with count-capping and variant support.

**Atom** &#8212; Canary | Import: `import { Badge } from '@arda-cards/design-system/canary'`

## Purpose

`Badge` is a thin Arda wrapper around the shadcn Badge primitive. It adds:

1. **Count mode** &#8212; pass `count={n}` to render a numeric badge. The value is capped at `max` (default 99); values above that display as `99+`. The badge receives `role="status"` when in count mode for live-region accessibility.
2. **Label mode** &#8212; pass text via `children` for string or label badges (e.g., `New`, `Draft`).

The component uses a rounded-md shape, tight padding, and 10 px semibold monospaced text to integrate cleanly with sidebar nav items, table cells, and page headers.

## Error Overlay Variant

The `error-overlay` variant is designed to be positioned absolutely over a `relative` container
(e.g., an image thumbnail). It uses destructive colors and a small fixed size suitable for
overlaying on grid cells or preview thumbnails.

## Accessibility

- When `count` is provided the badge receives `role="status"` so screen readers announce live updates.
- For purely decorative badges (e.g., a dot) use `aria-hidden="true"` on the badge element.
