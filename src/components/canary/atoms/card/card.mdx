## Purpose

`Card` provides a rounded (`rounded-xl`), bordered, shadowed surface for grouping related content. It is a pure composition surface &#8212; render any children inside it. Common uses include detail panels, dashboard widgets, settings sections, and kanban-style card previews.

Compose sub-components to build the desired layout:

```tsx
<Card>
  <CardHeader>
    <CardTitle>Inventory Summary</CardTitle>
    <CardDescription>Last updated 2 minutes ago</CardDescription>
    <CardAction>
      <Button variant="ghost" size="sm">Edit</Button>
    </CardAction>
  </CardHeader>
  <CardContent>
    {/* body content */}
  </CardContent>
  <CardFooter>
    <Button variant="outline">View all</Button>
  </CardFooter>
</Card>
```

## Accessibility

- `Card` renders as a `div`. If the card represents a landmark section, wrap it in an appropriate ARIA role (e.g., `<section aria-labelledby="card-title-id">`).
- `CardTitle` renders as a `div` for layout flexibility. Use a semantic heading (`h2`, `h3`, etc.) inside `CardTitle` when the card title should appear in the document outline.
