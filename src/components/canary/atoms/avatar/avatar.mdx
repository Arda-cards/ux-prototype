## Purpose

`Avatar` is built on Radix UI Avatar primitives. It renders an image when a valid `src` is available; when the image is absent or fails to load, `AvatarFallback` is shown instead (typically initials).

Compose the sub-components explicitly to control what appears in each slot:

```tsx
<Avatar size="lg">
  <AvatarImage src={user.avatarUrl} alt={user.name} />
  <AvatarFallback>{initials(user.name)}</AvatarFallback>
  <AvatarBadge />
</Avatar>
```

## entityName &#8212; Auto-computed initials

Pass `entityName` to `AvatarFallback` to have initials computed automatically via `getInitials`. This is an alternative to providing `children` manually; if both are present, `children` wins.

```tsx
<Avatar>
  <AvatarImage src={user.avatarUrl} alt={user.name} />
  <AvatarFallback entityName={user.name} />
</Avatar>
```

`getInitials` returns the first letter of up to two words, uppercased. An empty or whitespace-only name returns `?`.

## Accessibility

- Always provide a meaningful `alt` text on `AvatarImage` (e.g., the user's name).
- `AvatarFallback` content (initials or icon) is visible to screen readers when the image is absent; ensure it is meaningful or add `aria-label` on the parent `Avatar` if the fallback is purely visual.
- `AvatarBadge` is decorative by default; if the badge conveys status (e.g., online/offline), add an `aria-label` or `title` attribute.
