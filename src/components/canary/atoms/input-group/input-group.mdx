Compound input container with inline addons, prefix/suffix text, and action buttons.

## Purpose

`InputGroup` groups a text input or textarea with inline addons (icons, labels, currency symbols, action buttons). It renders as a single visually unified field with a shared border, focus ring, and error state rather than separate adjacent elements.

Compose sub-components to build the desired layout:

```tsx
// Leading currency symbol
<InputGroup>
  <InputGroupAddon align="inline-start">
    <InputGroupText>$</InputGroupText>
  </InputGroupAddon>
  <InputGroupInput placeholder="0.00" type="number" />
</InputGroup>

// Trailing action button
<InputGroup>
  <InputGroupInput placeholder="Search..." />
  <InputGroupAddon align="inline-end">
    <InputGroupButton>
      <SearchIcon />
    </InputGroupButton>
  </InputGroupAddon>
</InputGroup>
```

## Focus and error states

Focus ring and error styling are applied at the `InputGroup` level, not the individual input:

- **Focus**: When `InputGroupInput` or `InputGroupTextarea` receives `focus-visible`, the group border and ring update to use `ring` design tokens.
- **Error**: When the input carries `aria-invalid="true"`, the group border turns destructive.

---

## Accessibility

- `InputGroup` renders with `role="group"`. Pair it with an external `<label>` (via `htmlFor`) or `aria-label` / `aria-labelledby` on the group to give it an accessible name.
- Clicking anywhere in `InputGroupAddon` (other than embedded buttons) focuses the group's input control.
- Always provide `aria-label` or `aria-describedby` on the input for screen reader context.
