## Purpose

`ReadOnlyField` renders a stacked pair: a muted uppercase label above monospaced value text. It is the canonical display atom for named data fields in entity detail drawers, item panels, and supplier views.

When `value` is `undefined` or an empty string, the component shows a fallback string (default: em dash `&#8212;`). Custom rendering is supported via `children`, which takes precedence over `value` and `fallback`.

## Accessibility

- The label is rendered as a `span` with `text-muted-foreground` styling &#8212; it is not a `<label>` element, since this is a read-only display atom, not an interactive form field.
- For interactive form fields, use the shadcn `Label` + `Input` pattern or `InputGroup`.
- Value text is rendered in `font-mono` for numeric and code-like values to improve readability in dense layouts.
