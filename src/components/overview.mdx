# Components Section Overview

The component library is organized by track, then by level.

## Stable

`Components/Stable` is the default track for reusable components intended for broad usage today.

## Canary

`Components/Canary` is the canary track for components under active extraction/refinement.

Promotion path:

`Stable` represents what developers should reach for first. `Canary` is where extraction, replacement, and parity work happens before components are promoted into `Stable`.
