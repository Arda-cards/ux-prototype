# Components Section Overview

The component library is organized by track, then by level.

## Current

`Components/Current` is the default track for reusable components intended for broad usage today.

## Migration

`Components/Migration` is the migration track for components under active extraction/refinement.

Promotion path:

`Current` represents the baseline developers should reach for first. `Migration` is where extraction, replacement, and parity work happens before components are promoted into `Current`.
