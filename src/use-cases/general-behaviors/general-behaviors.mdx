# General Behaviors

Cross-cutting interaction patterns that apply to all entity types in the Arda platform. These use cases define the framework-level behaviors that every list view, detail panel, and form inherits.

## Relationship to Entity Use Cases

General behaviors define **what** a grid or panel can do. Entity-specific use cases (REF::ITM, REF::BA) define **which** columns, fields, and actions are available for each entity type. The entity use cases compose general behaviors with domain-specific configuration.
