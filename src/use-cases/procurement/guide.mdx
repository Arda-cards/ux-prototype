## Conventions

### Use Case Identifiers

Every use case and scenario has a structured identifier:

`PRO::AREA::NNNN::NNNN.TYPE`

- **PRO** &#8212; Procurement domain
- **AREA** &#8212; Functional area: `E2E` (End-to-End), `OQ` (Order Queue),
  `PO` (Purchase Orders), `RCV` (Receiving)
- **NNNN** &#8212; Use case number (first segment) and scenario number (second segment)
- **TYPE** &#8212; Scenario classification:
  - **US** &#8212; User Story (describes a user need or motivation)
  - **FS** &#8212; Functional Specification (describes system behavior or data rules)
  - **UC** &#8212; Use Case (describes a step-by-step interaction flow)

### Status Labels

Each use case group carries a status:

- **Covered** &#8212; Documented in at least one source specification and accounted for
  in the current design.
- **New-extends** &#8212; Extends an existing use case with additional scenarios not
  yet in the original specs.
- **New-distinct** &#8212; A wholly new use case identified during analysis.
- **Partial** &#8212; Some scenarios are covered; others require further specification.
