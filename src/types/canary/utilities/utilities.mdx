# Utilities

Plain TypeScript functions shared across canary components. These have no React
dependency and live under `src/types/canary/utilities/`.

---

## `getInitials`

Derives display initials from an entity name string. Splits on whitespace,
takes the first character of each word (up to two), and uppercases the result.

**Import:**

```ts
import { getInitials } from '@arda-cards/design-system/canary';
```

**Signature:** `getInitials(name: string): string`

## `getCroppedImage`

Canvas helper that applies crop coordinates, zoom factor, and rotation angle
to an image source and returns a `Blob`.

**Import:**

```ts
import { getCroppedImage } from '@arda-cards/design-system/canary';
```

**Signature:**

```ts
getCroppedImage(
  imageSrc: string,
  pixelCrop: PixelCrop,
  rotation?: number,       // degrees, default 0
  outputFormat?: string,    // MIME type, default 'image/jpeg'
  quality?: number,         // 0-1, default 0.85
): Promise<Blob>
```

## `image-upload-handlers`

Default handler implementations for `ImageUploadDialog`'s `onUpload` and `onCheckReachability`
props. These live in `src/types/canary/utilities/image-upload-handlers.ts`.

**Import:**

```ts
import {
  defaultUploadHandler,
  defaultReachabilityCheck,
} from '@arda-cards/design-system/canary';
```

### `defaultUploadHandler`

A mock upload handler intended for Storybook stories and development environments.

**Signature:** `(blob: Blob) => Promise<ImageUploadResult>`

**Behavior:** Waits approximately 1.5 seconds (simulating network latency) and then returns an `ImageUploadResult` containing a randomly selected picsum.photos URL. No actual file upload is performed &#8212; the `Blob` argument is ignored.

**Production note:** This handler must be replaced with a real implementation in production. It does not upload the image to any server.

### `defaultReachabilityCheck`

A mock reachability check intended for Storybook stories and development environments.

**Signature:** `(url: string) => Promise<boolean>`

**Behavior:** Returns `true` for all URLs except those containing the substring `"broken"`, which return `false`. No actual HTTP request is made. This allows story authors to simulate a broken-URL scenario by including `"broken"` anywhere in the URL string.

**Production note:** This handler must be replaced with a real implementation in production. A typical production implementation would perform a `HEAD` request (or similar lightweight probe) to verify that the URL resolves before staging the image.
